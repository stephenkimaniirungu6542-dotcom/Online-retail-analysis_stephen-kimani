Problem Statement

The real-world business challenge this project solves

The Business Problem

A UK-based online gift retailer operating across 38 countries had over half a million transaction records but no clear visibility into which customers were most valuable, which products drove the most revenue, what time of year performance peaked, or why 9,288 orders were being cancelled. The business was flying blind — unable to answer fundamental questions like: Who are our best customers? Where should we focus inventory? When do we need more staff?

The Solution

This project applies structured SQL querying, Python-based data cleaning and RFM modelling, and BI visualisation to transform raw transactional data into actionable intelligence — giving business decision-makers a clear picture of revenue drivers, customer segments, seasonal patterns, and product performance. Every insight in this report directly answers a real question a business manager or CEO would ask.

Data Cleaning & Preparation

SQL queries used to validate and clean the raw data before analysis

01 — Inspect raw data structure SQL 
-- Preview first 10 rows to understand the dataset
SELECT *
FROM `online_retail.transactions`
LIMIT 10;
02 — Check for nulls and data quality issues SQL 
-- Count missing values per column
SELECT
  COUNTIF(InvoiceNo IS NULL)    AS null_invoice,
  COUNTIF(CustomerID IS NULL)   AS null_customer,
  COUNTIF(Description IS NULL)  AS null_description,
  COUNTIF(Quantity <= 0)         AS zero_or_negative_qty,
  COUNTIF(UnitPrice <= 0)        AS zero_or_negative_price,
  COUNTIF(STARTS_WITH(InvoiceNo, 'C')) AS cancelled_orders
FROM `online_retail.transactions`;
03 — Create clean working table (remove nulls, cancellations, bad prices) SQL 
-- Clean table: only valid, completed, identified transactions
CREATE OR REPLACE TABLE `online_retail.clean_transactions` AS
SELECT
  InvoiceNo,
  StockCode,
  Description,
  Quantity,
  PARSE_DATETIME('%m/%d/%Y %H:%M', InvoiceDate) AS InvoiceDate,
  UnitPrice,
  CustomerID,
  Country,
  Quantity * UnitPrice AS Revenue
FROM `online_retail.transactions`
WHERE
  CustomerID IS NOT NULL
  AND Quantity > 0
  AND UnitPrice > 0
  AND NOT STARTS_WITH(InvoiceNo, 'C');

-- Result: 397,884 clean rows out of 541,909 raw rows

Python: Data Cleaning & EDA

Python code used for cleaning, exploration, and RFM analysis

data_cleaning.py — Load and clean the dataset PYTHON 
import pandas as pd
import numpy as np

# Load dataset (latin1 encoding handles special characters)
df = pd.read_csv('OnlineRetail.csv', encoding='latin1')

# Parse date column
df['InvoiceDate'] = pd.to_datetime(df['InvoiceDate'])
df['InvoiceNo'] = df['InvoiceNo'].astype(str)

# Create Revenue column
df['Revenue'] = df['Quantity'] * df['UnitPrice']

# Remove: nulls, cancellations, zero/negative quantities and prices
df_clean = df[
  (df['Quantity'] > 0) &
  (df['UnitPrice'] > 0) &
  (df['CustomerID'].notna()) &
  (~df['InvoiceNo'].str.startswith('C'))
].copy()

print(f"Raw rows: {len(df):,} | Clean rows: {len(df_clean):,}")
# Output: Raw rows: 541,909 | Clean rows: 397,884
rfm_analysis.py — RFM Customer Segmentation PYTHON 
# RFM: Recency, Frequency, Monetary — classic customer segmentation model
snapshot_date = df_clean['InvoiceDate'].max() + pd.Timedelta(days=1)

rfm = df_clean.groupby('CustomerID').agg(
    Recency  = ('InvoiceDate', lambda x: (snapshot_date - x.max()).days),
    Frequency= ('InvoiceNo',   'nunique'),
    Monetary = ('Revenue',      'sum')
).reset_index()

# Score each dimension 1–4 using quartiles
rfm['R_Score'] = pd.qcut(rfm['Recency'],   q=4, labels=[4,3,2,1])
rfm['F_Score'] = pd.qcut(rfm['Frequency'].rank(method='first'), q=4, labels=[1,2,3,4])
rfm['M_Score'] = pd.qcut(rfm['Monetary'],  q=4, labels=[1,2,3,4])

# Combine into RFM segment string (e.g. "444" = best customers)
rfm['RFM_Score'] = (
  rfm['R_Score'].astype(str) +
  rfm['F_Score'].astype(str) +
  rfm['M_Score'].astype(str)
)

# Champion customers: all 4s — recently bought, buy often, spend most
champions = rfm[rfm['RFM_Score'] == '444']
print(f"Champion customers: {len(champions)}")
# Output: Champion customers: 328

Revenue Analysis — SQL Queries

Structured queries that power the revenue insights in this report

04 — Monthly revenue trendSQL
SELECT
  FORMAT_DATE('%Y-%m', InvoiceDate)   AS Month,
  ROUND(SUM(Revenue), 2)             AS Monthly_Revenue,
  COUNT(DISTINCT InvoiceNo)          AS Total_Orders,
  COUNT(DISTINCT CustomerID)         AS Active_Customers
FROM `online_retail.clean_transactions`
GROUP BY Month
ORDER BY Month;
05 — Revenue by country (Top 10)SQL
SELECT
  Country,
  ROUND(SUM(Revenue), 2)    AS Total_Revenue,
  COUNT(DISTINCT CustomerID) AS Customers,
  ROUND(SUM(Revenue) * 100.0 /
    SUM(SUM(Revenue)) OVER(), 2) AS Revenue_Share_Pct
FROM `online_retail.clean_transactions`
GROUP BY Country
ORDER BY Total_Revenue DESC
LIMIT 10;
06 — Top 10 revenue-generating productsSQL
SELECT
  StockCode,
  Description,
  SUM(Quantity)              AS Units_Sold,
  ROUND(AVG(UnitPrice), 2)  AS Avg_Price,
  ROUND(SUM(Revenue), 2)   AS Total_Revenue
FROM `online_retail.clean_transactions`
GROUP BY StockCode, Description
ORDER BY Total_Revenue DESC
LIMIT 10;
07 — Customer RFM segmentation in SQLSQL
WITH rfm_base AS (
  SELECT
    CustomerID,
    DATE_DIFF('2011-12-10', MAX(DATE(InvoiceDate)), DAY) AS Recency,
    COUNT(DISTINCT InvoiceNo)                              AS Frequency,
    ROUND(SUM(Revenue), 2)                                 AS Monetary
  FROM `online_retail.clean_transactions`
  GROUP BY CustomerID
),
rfm_scored AS (
  SELECT *,
    NTILE(4) OVER (ORDER BY Recency DESC)   AS R,
    NTILE(4) OVER (ORDER BY Frequency)         AS F,
    NTILE(4) OVER (ORDER BY Monetary)          AS M
  FROM rfm_base
)
SELECT *,
  CONCAT(CAST(R AS STRING), CAST(F AS STRING), CAST(M AS STRING)) AS RFM_Segment
FROM rfm_scored
ORDER BY Monetary DESC;
08 — Peak shopping hoursSQL
SELECT
  EXTRACT(HOUR FROM InvoiceDate) AS Hour_of_Day,
  COUNT(DISTINCT InvoiceNo)      AS Number_of_Orders,
  ROUND(SUM(Revenue), 2)        AS Revenue
FROM `online_retail.clean_transactions`
GROUP BY Hour_of_Day
ORDER BY Number_of_Orders DESC;
09 — Customer repeat purchase rateSQL
SELECT
  COUNT(DISTINCT CustomerID)                             AS Total_Customers,
  COUNTIF(order_count > 1)                               AS Repeat_Customers,
  ROUND(COUNTIF(order_count > 1) * 100.0 /
    COUNT(*), 1)                                          AS Retention_Rate_Pct
FROM (
  SELECT CustomerID, COUNT(DISTINCT InvoiceNo) AS order_count
  FROM `online_retail.clean_transactions`
  GROUP BY CustomerID
);
-- Result: 65.6% retention rate (2,845 of 4,338 customers bought more than once)

Key Findings

What the data actually tells us — translated into business language

📈

Q4 Revenue Surge

November 2011 was the single highest revenue month at £1.16M — 103% above the annual monthly average. This is driven by pre-Christmas bulk gifting orders.

🌍

UK Revenue Concentration Risk

The United Kingdom accounts for 82% of total revenue (£7.3M). While strong, this over-reliance on a single market is a business risk if UK demand slows.

👥

Strong Customer Retention

65.6% of customers made more than one purchase during the year — indicating a loyal repeat base that can be targeted with loyalty programmes and upselling.

⏰

Midday Peak Ordering

12 PM sees the highest order volume with 3,130 orders. Customer service, warehouse staffing, and server capacity should be optimised for the 10 AM–2 PM window.

❌

High Cancellation Volume

9,288 orders were cancelled — representing a significant revenue leakage risk. Root causes need to be investigated: stock-outs, payment failures, or poor UX.

🎯

Champion Customer Segment

328 customers scored RFM 444 — they bought recently, buy frequently, and spend the most. These customers deserve a dedicated retention and VIP strategy.

📦

Bulk Gifting Products Dominate

Top revenue products are gift and homeware items often ordered in large quantities. This suggests a B2B wholesale segment worth developing further.

🌱

International Growth Opportunity

Netherlands, Germany, and France collectively contribute £723K with far fewer customers than the UK — suggesting high average order values and untapped international growth potential.

Business Recommendations

Actionable next steps for the business based on the analysis

1

Investigate and reduce cancellations — Priority: High

9,288 cancellations need a root cause analysis. Build a SQL query to identify cancellations by product, country, and time of year. Even reducing cancellations by 20% would recover an estimated £300K+ in revenue.

2

Launch a VIP retention programme for Champion customers — Priority: High

The 328 RFM 444 customers are your highest-value segment. A dedicated loyalty programme, early access to new products, or personalised offers would increase their lifetime value significantly.

3

Build Q4 inventory and staffing strategy around the data — Priority: Medium

Revenue spikes 71% between August and November. The business should pre-stock top products by September and scale warehouse and customer service staff ahead of the November peak.

4

Develop international markets — Priority: Medium

Netherlands, Germany, and France customers spend more per order than UK customers on average. A targeted international marketing campaign with localised content could double revenue from these markets within 12 months.

5

Resolve anonymous orders (135,080 missing CustomerIDs) — Priority: Medium

Over 135,000 transactions have no CustomerID, making it impossible to track their behaviour or re-target them. Introducing mandatory account creation or email capture at checkout would unlock this data for future analysis.

6

Explore B2B wholesale channel — Priority: Low-Medium

The top revenue products (cakestands, craft paper, storage jars) are ordered in very high quantities — suggesting resellers. A formal wholesale programme with volume pricing could formalise and grow this segment.
